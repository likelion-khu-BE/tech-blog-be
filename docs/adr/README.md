# Architecture Decision Records (ADR)

프로젝트에서 내린 주요 결정과 "왜 그렇게 했는지"를 기록하는 곳.

코드를 보면 **뭘** 했는지는 알 수 있다. 근데 **왜** 그렇게 했는지는 안 보인다.
3개월 뒤에 "이거 왜 이렇게 한 거지?" 하는 순간이 올 때 여기를 보면 된다.

## 규칙

- ADR은 한번 쓰면 **고치지 않는다** (불변). 결정 당시의 맥락을 보존하는 게 핵심.
- 결정을 바꾸고 싶으면? 새 ADR을 쓰고, 기존 건 Status를 `Superseded by [XXXX]`로 바꾼다.
- 번호는 `0001`, `0002`, ... 순서대로.
- 템플릿: [template.md](template.md)
